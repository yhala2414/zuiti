# 申请添加全局变量申请表

## 目的

用于约束哪些视觉参数应沉淀到 `app/globals.css`，避免把单页装饰、单组件私有配色或一次性偏移量错误上升为全局 token。

## 准入规则

- 同一视觉值已在 2 个及以上页面或组件重复出现，且语义一致。
- 该值属于设计系统基础能力，例如颜色、圆角、阴影、间距、动效时长、主题渐变。
- 该值会影响 `antd-mobile` 主题覆写、全局基类或多个复用组件。
- 抽象后变量名能够表达语义，而不是描述某个业务页面。

## 不进入全局变量的情况

- 只服务单个页面英雄区或单个插画的渐变、阴影、异形圆角。
- 只用于装饰图标的局部颜色组合，例如 `.zuiti-icon-*`、结果卡片图标的拟物细节。
- 只为某个断点、某个绝对定位补偿存在的临时偏移。
- 数值虽相近但视觉语义不同，强行复用会削弱后续调优空间。

## 申请表模板

| 字段 | 填写说明 |
| --- | --- |
| 变量名 | 使用语义化命名，例如 `--radius-pill`、`--text-secondary` |
| 类别 | 颜色 / 阴影 / 圆角 / 间距 / 动效 / 渐变 / 组件主题 |
| 当前出现位置 | 列出重复来源，至少 2 处 |
| 建议值 | 直接写 CSS 值 |
| 建议作用域 | `app/globals.css` / 页面局部变量 / 组件局部变量 |
| 预期消费者 | 受影响的页面、组件或全局类 |
| 变更风险 | 是否可能造成串色、层级变化、动画节奏变化 |
| 结论 | 通过 / 暂缓 / 保持局部 |

## 建议保持局部变量或局部常量的项

| 项目 | 原因 | 建议作用域 |
| --- | --- | --- |
| 首页英雄区光斑、背景渐变、拟物插画高光 | 强页面特征，跨页不可复用 | 页面级 |
| `.zuiti-icon-*` 的颜色、阴影、异形圆角 | 属于单图标绘制参数，改成全局 token 会稀释语义 | 全局类内部私有值 |
| `ResultCard` 内微信 / 邮件 / 表情图标色板 | 仅服务结果卡片图标，不具备全局主题意义 | 组件级 |
| 语气页预览卡的 blob 异形圆角与定位偏移 | 明显依赖单页面构图 | 页面级 |
| 响应式断点下的补偿 padding / gap / absolute offset | 是布局策略，不是设计令牌 | 页面级或组件级 |

## 本次落地结论

- 已进入 `app/globals.css` 的变量，应优先服务基础文本层级、玻璃表面、通用圆角、共享动效和滑杆主题。
- 仍保留在页面或组件内的值，主要是插画细节、单页装饰和业务语义强绑定样式。
- 后续若新增变量，需先用本表登记重复来源与语义，再决定是否进入全局治理。
